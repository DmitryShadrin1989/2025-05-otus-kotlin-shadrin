# Функциональные требования к системе «Электронная записная книжка»


## 1. Аутентификация и управление учётной записью


1.1. **Регистрация пользователя**
- **Источник требования:** конечные пользователи.
- **Описание требования:** реализовать функционал регистрации с обязательными полями:
    - электронная почта (валидация формата);
    - пароль (минимум 8 символов, требование сложности);
    - имя пользователя (опционально).
- **Приоритет требования:** высокий.
- **Критерии приёмки:**
    - система проверяет уникальность email;
    - при успешной регистрации отправляется подтверждающее письмо;
    - пользователь перенаправляется в личный кабинет после подтверждения.

1.2. **Вход в систему**
- **Источник требования:** зарегистрированные пользователи.
- **Описание требования:** обеспечить аутентификацию по email и паролю с возможностью «запомнить меня».
- **Приоритет требования:** высокий.
- **Критерии приёмки:**
    - при неверных данных выводится сообщение об ошибке;
    - после входа пользователь попадает на страницу со списком записей;
    - сессия сохраняется согласно настройкам «запомнить меня».

1.3. **Восстановление пароля**
- **Источник требования:** пользователи, забывшие пароль.
- **Описание требования:** реализовать механизм сброса пароля через email.
- **Приоритет требования:** средний.
- **Критерии приёмки:**
    - пользователь вводит email и получает ссылку для сброса;
    - ссылка действует 1 час;
    - после смены пароля приходит уведомление.

## 2. Управление записями

2.1. **Создание записи**
- **Источник требования:** конечные пользователи.
- **Описание требования:** предоставить форму создания записи с полями:
    - заголовок (до 100 символов);
    - текст записи (неограниченный объём);
    - категория (выбор из списка);
    - дата/время напоминания (опционально).
- **Приоритет требования:** высокий.
- **Критерии приёмки:**
    - запись сохраняется в БД;
    - пользователь видит запись в списке сразу после создания;
    - поля проверяются на корректность ввода.

2.2. **Редактирование записи**
- **Источник требования:** пользователи, желающие изменить записи.
- **Описание требования:** разрешить изменение всех полей записи кроме даты создания.
- **Приоритет требования:** высокий.
- **Критерии приёмки:**
    - после сохранения изменения отображаются в списке;
    - история правок не сохраняется (простая замена).

2.3. **Удаление записи**
- **Источник требования:** пользователи, желающие удалить записи.
- **Описание требования:** реализовать мягкое удаление (в корзину) с возможностью восстановления в течение 7 дней.
- **Приоритет требования:** средний.
- **Критерии приёмки:**
    - удалённая запись перемещается в раздел «Корзина»;
    - из корзины запись можно восстановить или окончательно удалить;
    - через 7 дней записи из корзины удаляются автоматически.


## 3. Поиск и фильтрация


3.1. **Поиск по записям**
- **Источник требования:** пользователи, ищущие конкретные записи.
- **Описание требования:** реализовать поиск по:
    - заголовку;
    - тексту записи;
    - категории.
- **Приоритет требования:** средний.
- **Критерии приёмки:**
    - результаты отображаются в реальном времени;
    - при отсутствии совпадений выводится сообщение «Записей не найдено».

3.2. **Фильтрация по категориям**
- **Источник требования:** пользователи, организующие записи.
- **Описание требования:** позволить фильтровать записи по выбранным категориям.
- **Приоритет требования:** средний.
- **Критерии приёмки:**
    - список категорий отображается в боковом меню;
    - при выборе категории показываются только соответствующие записи;
    - есть кнопка «Все записи» для сброса фильтра.

## 4. Напоминания и уведомления

4.1. **Установка напоминаний**
- **Источник требования:** пользователи, нуждающиеся в напоминаниях.
- **Описание требования:** разрешить установку напоминаний с выбором:
    - даты;
    - времени;
    - способа уведомления (push, email).
- **Приоритет требования:** высокий.
- **Критерии приёмки:**
    - напоминание срабатывает за 15 минут до указанного времени;
    - в списке записей отображается значок напоминания.

4.2. **Управление уведомлениями**
- **Источник требования:** пользователи, настраивающие оповещения.
- **Описание требования:** предоставить настройки уведомлений:
    - включение/отключение push;
    - выбор времени отправки email.
- **Приоритет требования:** средний.
- **Критерии приёмки:**
    - настройки сохраняются между сеансами;
    - изменения применяются немедленно.

## 5. Управление категориями

5.1. **Создание категории**
- **Источник требования:** пользователи, организующие записи.
- **Описание требования:** разрешить добавление новых категорий с названием и цветом метки.
- **Приоритет требования:** средний.
- **Критерии приёмки:**
    - категория появляется в списке сразу после создания;
    - цвет метки сохраняется.

5.2. **Редактирование/удаление категории**
- **Источник требования:** пользователи, управляющие категориями.
- **Описание требования:** позволить изменять название и цвет категории, а также удалять её.
- **Приоритет требования:** низкий.
- **Критерии приёмки:**
    - при удалении категории записи переходят в «Без категории»;
    - изменения отображаются мгновенно.
